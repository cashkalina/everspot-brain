# DepositBatch

## Overview

DepositBatch groups physical payment transactions — typically cash and check payments — into a single bank deposit record for reconciliation purposes. When a cemetery closes out a day's receipts or prepares a trip to the bank, those transactions are collected into a DepositBatch, which then moves through a `pending` → `posted` lifecycle.

Each batch has a `date` (the deposit date) and an optional `settlement_date` (when the bank confirms the funds). The `amt` column tracks the total batch amount in cents via [HasMoneyFields](../../../system/traits/index.md#hasmoneyfields). A `model_no` provides a human-readable batch identifier. Two batch types are defined: `manual` (prepared by staff) and `automatic` (generated by the system, e.g. for ACH settlement batches).

Individual transactions are linked back to their batch via the `deposit_batch_id` FK on the `transactions` table. DepositBatch itself defines no outbound relationships — navigation from a batch to its transactions is achieved via `Transaction::where('deposit_batch_id', $batch->id)` or via the reverse of Transaction's `depositBatch()` relationship.

**Note:** DepositBatch has no observer registered in either `EventServiceProvider` or `TransactionServiceProvider`.

**Foreign keys:** `created_by`, `updated_by`, `deleted_by` → `users.id`

**Indexes:** Unique index on `model_no`; single-column index on `status`; FK-backing indexes on `created_by`, `updated_by`, `deleted_by`.

## Common Usage

```php
// Create a manual deposit batch
$batch = DepositBatch::create([
    'date'   => today(),
    'status' => 'pending',
    'type'   => 'manual',
    'amt'    => 50000, // $500.00 in cents
]);

// Post a deposit batch
$batch->update(['status' => 'posted', 'settlement_date' => today()->addDays(2)]);

// Get all transactions in a batch
$transactions = Transaction::where('deposit_batch_id', $batch->id)->get();

// Find pending batches
$pending = DepositBatch::where('status', 'pending')->get();

// Format the batch type for display
echo $batch->formatted_type; // "Manual"
```
