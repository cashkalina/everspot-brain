# PaymentMethodRequest

## Overview

PaymentMethodRequest represents a one-time, token-authenticated request for a customer to add a new payment method. When cemetery staff want a customer to store a card or bank account, they create a PaymentMethodRequest; the system generates a secure token and an expiry, then either displays an in-person form immediately or sends the customer a link via email.

Two delivery methods are supported: `in-person` (display the form right now at the front desk) and `email` (send a link the customer completes at their own pace). The `token` field contains a cryptographically random hex string (64 chars, 32 bytes) generated by the observer on creation; this token is embedded in a publicly-accessible URL so the customer can fill in their payment details without needing to log in.

The model tracks the full lifecycle of the request: when it was visited (`visited_at`), when it was completed (`completed_at`), and when it expires (`expires_at`, defaulting to 24 hours). For ACH payments that require micro-deposit verification, `next_action` is set to `'microdeposits'` and `microverify_sent_at`/`microverify_attempts` track the verification state. The `payment_method_type` column is cast to a `PaymentMethodType` enum (`credit-card` or `ach`) to constrain the type of method accepted.

Sensitive fields (`referer_url`, `success_url`, `token`) are excluded from reporting via `$disabledReportColumns`.

## Scopes

- `usable(Builder $query)` ‚Äî filters to requests where `completed_at IS NULL` and `expires_at > now()`; i.e. not yet completed and not yet expired
- `recentlyCompleted(Builder $query)` ‚Äî filters to requests completed within the last 14 days (`completed_at > now() - 14 days`)

## Observers

- `PaymentMethodRequestObserver` ‚Äî registered in `TransactionServiceProvider::registerObservers()` (`PaymentMethodRequest::observe(PaymentMethodRequestObserver::class)`). Handles:
  - `creating` ‚Äî sets `token` (64-char hex from `bin2hex(random_bytes(32))`), `expires_at` (now + 24 hours), and `success_url` (in-person: referer URL + `?customer_id=‚Ä¶`; email: named route `app.external.payments.add-method-success`)
  - `created`, `updated`, `deleted`, `restored`, `forceDeleted` ‚Äî no-op stubs

## Key Methods

- `setCompleted(): void` ‚Äî marks the request as completed by setting `completed_at` to now via `update()`
- `isCompleted(): bool` ‚Äî true if `completed_at` is not null
- `isExpired(): bool` ‚Äî true if `expires_at` is in the past
- `isPending(): bool` ‚Äî true if not completed, not expired, and not yet visited (`visited_at` is null)
- `isInProgress(): bool` ‚Äî true if not completed, not expired, visited, and no `next_action` pending
- `requiresMicrodeposits(): bool` ‚Äî true if `next_action = 'microdeposits'` and not completed
- `canBeCompletedByStaff(): bool` ‚Äî true if not completed and not expired (staff can manually complete)
- `getExternalUrl(): string` ‚Äî returns the publicly-accessible URL for this request: the microdeposit verification route if microdeposits are required, otherwise the add-method route, both keyed by `token`

## Common Usage

```php
// Create an in-person payment method request
$request = PaymentMethodRequest::create([
    'customer_id'          => $customer->id,
    'user_id'              => auth()->id(),
    'method'               => 'in-person',
    'payment_method_type'  => 'credit-card',
    'referer_url'          => request()->headers->get('referer'),
]);
// Observer auto-sets token, expires_at, and success_url

// Create an email request
$request = PaymentMethodRequest::create([
    'customer_id'         => $customer->id,
    'user_id'             => auth()->id(),
    'method'              => 'email',
    'payment_method_type' => 'ach',
    'referer_url'         => url()->current(),
]);

// Get the URL to present or email to the customer
$url = $request->getExternalUrl();

// Check status
if ($request->isPending()) {
    // Not yet visited
}
if ($request->requiresMicrodeposits()) {
    // ACH microdeposit verification pending
}

// Mark as completed (e.g. after the payment processor callback)
$request->setCompleted();

// Find all usable (active) requests for a customer
$active = PaymentMethodRequest::usable()
    ->where('customer_id', $customer->id)
    ->get();
```
