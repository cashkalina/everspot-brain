# Attribute

## Overview

The Attribute model defines a single custom-attribute definition in the Everspot EAV (Entity–Attribute–Value) system. Each Attribute record describes one kind of data that can be attached to any entity (e.g. Customer, Interment) that uses the [HasAttributes](../../../system/traits/index.md#hasattributes) trait. The definition includes the attribute's label, a machine-readable `key` (auto-generated by slugifying the label), the data `type` (e.g. `string`, `date`, `list-option`, `model-reference`), optional validation rules, a default value, and a `config_data` blob that carries type-specific configuration through the `AttributeConfigCast`.

Attributes are organised into a two-level self-referential hierarchy via `parent_attribute_id`: parent attributes define high-level groupings (such as collection or array attributes), while child attributes represent individual fields within those groupings. An attribute can be soft-deleted when it is retired, preserving historical values. The `TYPES` constant is the authoritative catalogue of supported data types and their UI labels and input-type hints.

Attributes are not directly attached to model instances; that linkage is managed by [EntityAttribute](./entity-attribute.md), which binds an Attribute definition to a specific entity class and area. Actual stored values live in [AttributeValue](./attribute-value.md).

## Accessors & Mutators

- `getFormattedTypeAttribute(): mixed` — returns the human-readable label for the current `type` from `TYPES[$this->type]['formatted']`
- `getConfigAttribute(): ?AttributeConfigInterface` — returns `$this->config_data` (the cast AttributeConfigInterface object); provides a named accessor for the cast column
- `setKeyAttribute(): void` — **mutator**: auto-sets `key` to `Str::slug($this->label)` whenever the attribute is set

## Traits

- [SoftDeletes](../../../system/traits/index.md#softdeletes) — attribute definitions are soft-deleted, never hard-deleted

## Relationships

- `parentAttribute()` — belongs to [Attribute](./attribute.md) (`parent_attribute_id`): the parent attribute in a hierarchical grouping
- `childAttributes()` — has many [Attribute](./attribute.md) (`parent_attribute_id`): child attributes that belong to this parent
- `entityAttributes()` — has many [EntityAttribute](./entity-attribute.md) (`attribute_id`): the entity-level bindings that reference this attribute definition

## Common Usage

```php
// Look up what types can be created by users
$creatableTypes = collect(Attribute::TYPES)
    ->filter(fn($t) => $t['enable_create'])
    ->keys();

// Create a new text attribute
$attr = Attribute::create([
    'type'  => 'string',
    'label' => 'Member Since',
    // 'key' is auto-set to 'member-since' via the mutator
]);

// Retrieve the human-readable type label
echo $attr->formatted_type; // "Text"

// Tree traversal
$children = $attr->childAttributes;
$root     = $attr->parentAttribute;

// Fetch all entity-attribute bindings using this definition
$bindings = $attr->entityAttributes()->with('attributeArea')->get();
```
