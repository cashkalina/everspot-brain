# scripts/LIBRARY.md — the §11.2 7b script-library index

**Purpose.** The curated, versioned, **tested**, dataset-agnostic library of cleansing
primitives + validators. Code is **never loaded as prose** — this index is how the
orchestrator (and the `profile` / `map` / `cleanse` stages) know what exists.

**The rule (binding):**

- `profile` / `map` / `cleanse` reach for this library **BEFORE inventing** any transform.
- A new data shape that demands a new utility **adds it here WITH a golden test** (`tests/unit/`), so the next dataset benefits automatically.
- Library code is **general only** — no client-specific column names or value maps (those live in a project's ledger/mapping).
- Each utility is **version-stamped** (`VERSION` in its module). The stamp feeds the Tier-3 transform cache (§5.3) — bumping a version lazily recomputes only re-encountered strings.

## Cleansing primitives

Each returns a `cellcontract.Cell` (`{value, confidence, method, needs_llm, reason}`).
The Cell is the contract; an unparseable / ambiguous value routes to the LLM tier via
`needs_llm=True` — it is **never** silently dropped.

| Utility | Module | Version | Library | Purpose |
|---|---|---|---|---|
| `parse_name` | `parse_name.py` | 1.0.0 | `nameparser` | Person-name cell → `{title, first, middle, last, suffix, nickname}`. Detects/un-reverses `"Last, First"`; flags **two-people-in-one-cell** → `needs_llm` (no auto-split). |
| `normalize_date` | `normalize_date.py` | 1.0.0 | `dateutil` | Date cell → partial date `{year, month, day, estimated}`. Flags circa/estimated, DD-MM vs MM-DD ambiguity, 2-digit-year century ambiguity, unparseable. |
| `normalize_phone` | `normalize_phone.py` | 1.0.0 | `phonenumbers` | Phone cell → digits-only (E.164-ish). Multi-number / extension / vanity → low-confidence `needs_llm`. |
| `standardize_address` | `standardize_address.py` | 1.0.0 | `usaddress` (libpostal optional, lazy) | Freeform US address → `{line_one, line_two, city, state, zip}`. Confidence reflects tag coverage; ambiguous/garbage → `needs_llm`. |
| `to_cents` | `to_cents.py` | 1.0.0 | `decimal.Decimal` | Money string → **integer cents**, exact (never float). Handles currency symbols + parens-negatives. Non-numeric free text → `needs_llm` to *extract* (never LLM-arithmetic). |
| `digits_only` | `digits_only.py` | 1.0.0 | stdlib `re` | Strip everything but digits (ID-like fields). Always confident; never routes to the LLM. |
| `resolve_list_option` | `resolve_list_option.py` | 1.0.0 | `rapidfuzz` | Raw code/label → a **real tenant `list_option` id** from the Wave-0 snapshot. Exact → confident; ≥90 → medium; 75–90 → low; unknown → `needs_llm` (a value-set question). **Never invents an id.** |

## Validators

| Validator | Module | Version | Purpose |
|---|---|---|---|
| `validate_canonical` / `validate_run` | `validate.py` | 1.0.0 | Stage 10 — validate a canonical record set: schema (via `_make_ref_validator`, the `referencing`-based `$ref` resolver — NOT the deprecated `RefResolver`, L4) + `contract.validate_record` conformance, **referential integrity** (every `*_ref` resolves to an existing record of the right entity; `cemetery`/`list_option`/`property_type` refs are Wave-0b prereqs, not dangling), unresolved value-set `*_id` warnings, **count conservation** (manifest-vs-disposition via `reconcile.conserve` — `unexplained_dropped` BLOCKING, H1), and a low-confidence field sample. On a **scoped** v2 run (detected via `snapshots/<v>/delta.json`) seeds the ext→entity index from `ledger/external_ids.json` so a ref to an already-loaded unchanged parent is satisfied (M2). Returns a PASS/FAIL gate (dangling FK / missing-required / contract / unexplained-drop = BLOCKING; low-confidence / unresolved-ref = WARNING). GENERAL — keys off the contract + canonical schema, no client columns. |

Calendar + partial-date-contract validity currently lives **inside the assembler**
(`assemble._compose_partial_date` — calendar validity + "day requires month"; see
`knowledge/topics/partial-dates.md` and LESSONS #4). When a validity rule recurs and
generalizes, it **graduates** into a standalone, independently-tested library validator
here (recorded in `CHANGELOG.md`) — that graduation is the §11.3 anti-bloat mechanism in
action.

## Profilers & reconcilers (stage utilities)

| Utility | Module | Version | Purpose |
|---|---|---|---|
| `profile_snapshot` / `profile_table` | `profile.py` | 1.0.0 | Stage 3 — per-column stats (non-null/distinct/blank/dtype/min-max/sample — **PII-signalled columns' samples + value-set keys are redacted to a stable `[redacted:<sha256>]` token**, not just count-capped, L1), candidate keys (single + minimal-unique composites + **natural-key completion** of near-unique business locators, each with a uniqueness ratio + quality score; the composite search is BOUNDED — top-K(15) by cardinality, distinct-product short-circuit, a hard combo budget, and a `candidate_keys_truncated` flag when bounded, M6), value-set candidates (low-cardinality columns → value→freq map, with a `truncated` flag), and data-shape **signals** (name/joint_name/date/split-date/phone/money/address/coordinate/two_digit_year) → `knowledge/INDEX` trigger tags. | `map_draft` / `compute_drafts` | `map_draft.py` | 1.0.0 | Stage 5 — **deterministic auto-draft mapping**. Per source column proposes a target `entity.field` + action via name/shape match against the **contract** logical fields + profile signals (name→customer names; split Y/M/D→partial-date field; coded column matching a `list_option` binding→value_map). Resolves value-sets to **real tenant ids** via `resolve_list_option` (never invents). Exact + complete value sets→recorded default at confidence 1.0; a value resolved only via a **non-exact fuzzy (≥90) match** stays a `value_set` gap at confidence <1.0, and a profiler-**truncated** value set emits a gap (unseen codes ⇒ completeness can't be claimed) — L6. Ambiguous / unmapped-feeding-required / required-with-no-source / unresolved value→a **gap** (question). DEFERS the multi-entity routing of a flat row (`secondary_entities`) — emits it as a structural gap. Never clobbers a settled ledger (`.draft` sidecar). Writes `mapping.yaml` + `value_sets.yaml` + `runs/<v>/mapping_review.md`. GENERAL — no client column names. |
| `discover` | `discover.py` | 1.0.0 | Stage 6 + **THE QUESTION ROUND** — one batched pass (profile + draft + DRY assemble + validate) aggregating ALL undecidable subjects into `runs/<v>/questions.json` (conforms to `schemas/question.schema.json` + §9.4) + `questions.md` (rendered FROM json). Applies the ask-policy: clean value-set/obvious default→`auto-resolved` (logged); unresolvable value / missing source_key / required-with-no-source / blocking validation→`open`. **Idempotent (§9.5):** stable subject-derived ids (`q_<kind>__<subject>`); a subject settled in the on-disk mapping/value_sets or an answered `ledger/questions/<id>.json` is never re-asked. `_dedupe_by_id` raises a loud collision if two DISTINCT subjects (`Question.subject_key`) share an id rather than silently dropping the OPEN one (L5). Exposes `any_open` / exit 1 as the orchestrator gate. GENERAL. |
| `apply_answers` | `answer.py` | 1.0.0 | THE QUESTION ROUND apply step — reads `answers.json` (`{id: answer}`) or `accept-all` (every `proposed_answer`); persists each to `ledger/questions/<id>.json` (status + rationale); writes answers back into `mapping.yaml`/`value_sets.yaml` (value-set ids token-keyed for the value_map chain; source_key→project source decl) so the next `assemble` picks them up. A value_set answer containing ANY `None` is kept OPEN, never accept-all'd (M1, §9.2); a skip REQUIRES a rationale (bare `"skip"` rejected → stays OPEN). Exposes `any_open`, `skipped_blocking`, and `gate_clear` (`not any_open and skipped_blocking == 0` — the real orchestrator gate, L3). GENERAL. |

| `reconcile_offline` / `reconcile_live` | `reconcile.py` | 1.1.0 | Stage 10/12 — source→canonical count conservation per entity (offline, never networks, now via `conserve`) and canonical↔live-tenant (`--live`, the only network path). `--live` does TWO lenses: (1) **count conservation** by external_id presence, and (2) **field-level VALUE** check (SPEC §15.3): bulk-fetches each entity resource (all cols incl. partial-date `_year/_month/_day/_estimated`) + the external-ids map + property `attribute-values` in O(pages, no per-record calls), then projects each canonical record via the loader's OWN oracle (`orion_load.project_payload` / `project_location`) and diffs field-by-field against the live row. ## Adding a utility (checklist)

1. New `<name>.py` with a docstring (purpose + LLM-fallback triggers), a `VERSION` stamp, and a `Cell`-returning `clean(...)`.
2. A golden test in `tests/unit/` (golden inputs → expected outputs).
3. A row in this table.
4. General only — if it needs a client's column name or value map, it belongs in the ledger, not here.
